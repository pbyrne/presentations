Seriously, What is a Version Control System?

  • It keeps track of your things.
  • It remembers the changes that have been made, and by whom.
  • It lets you go back and forth through the history.

Thesaurus

Thesaurus

  • repository: the history of changes (aka: repo)
  • working copy: the set of files being worked on
  • commit: a particular set of changes (aka: revision, changeset)
  • branch: a diverging set of changes
  • tag: a bookmark to a particular commit
  • index: a staging area with items to be committed
  • pull request: a request to merge your changes into another branch or repository
  • check out: to update your working copy to a particular branch, revision, or tag
  • commit: to save your changes to your repository
  • merge: to apply someone's changes to a branch
  • push: to send your changes to another repository
  • pull: to download changes from another repository

Warning!

Working with the Central Repository

  • git clone http://example.com/foo.git: Set up a local repository with all of the changes at http://example.com/foo.git
  • git push: Push all of your changes to the default central repository (alternatively: git push origin master to specifically push the master branch to the server named origin)
  • git pull: Pull the changes from the default central repository and merge them into your working copy

Working with your Working Copy

  • git status: Display details of the working copy, such as new, deleted, or modified files
  • git add foo.txt: Add foo.txt to the index
  • git diff: See the changes that you've made, but have not added to the index or committed
  • git diff --cached: See the changes that you've added to the index, but haven't committed
  • git checkout foo.txt: Undo the changes to foo.txt and replace with the version from the repository

Working with Branches

  • git branch: Display list of local branches with the current branch highlighted
  • git branch --all: Also list the branches on the remote server
  • git checkout bar: switch your working copy to the branch bar
  • git diff otherbranch: See the changes between your current branch and otherbranch
  • git merge otherbranch: Apply the changes in otherbranch into your branch

Common Workflow

  • git pull to make sure you have everyone's changes
  • Work on stuff, make some changes of your own
  • git status to get an overview of what you've done
  • git diff to review the specific changes that you've made
  • git add . to add your changes to the index
  • git commit -m "Made the thing more awesome" to commit your change to your local repo
  • git push to push your changes to the central repo so everyone knows how awesome you are
